# C-034 — Intent Map: Policy Modification

**INT-034-01: Report Policy Conflict**
- **Trigger**: `POLICY` finding
- **Outcome**: Structured conflict attached to intervention

**INT-034-03: Modify Existing Policy**
- **Trigger**: `action=MODIFY_POLICY`
- **Outcome**: `DecisionProposal` generated; `persistProposal` gate activated.

**INT-034-04: Register New Policy**
- **Trigger**: `action=REGISTER_POLICY`
- **Outcome**:
  - `DecisionProposal` generated (new policy draft intent)
  - `persistProposal` gate activated
  - Policy root allocation deferred to PRD-033 at Registration Execution time

**INT-034-05: Approval Flow**
- **Trigger**: User `APPROVE`
- **Outcome**: `PersistSession` commits `DecisionVersion`.

**INT-034-07: No Policy Change Acknowledged**
- **Trigger**: `action=KEEP_POLICY`
- **Outcome**:
  - `AuditSink` records `POLICY_ACKNOWLEDGED` (policy acknowledgement record)
  - `DecisionVersion` 생성 없음
  - 정책 상태 유지
  - POLICY_ACKNOWLEDGED MUST NOT reuse SAFETY block record types (BLOCK_KEY / INFORMED_BLOCK / MIN_PERSIST).

Additional Constraint:
- KEEP_POLICY MUST NOT alter any execution control flow.
- KEEP_POLICY MUST NOT influence PersistSession eligibility.
- KEEP_POLICY MUST NOT enqueue PolicyRegistrationRequest.
- KEEP_POLICY MUST be execution-neutral.

**INT-034-08: Policy Registration Eligibility Created**
- **Trigger**:
  - `PersistSession` successfully commits a `DecisionVersion`
  - AND `Decision.scope` matches `policy.*`
- **Outcome**:
  - A `PolicyRegistrationRequest` is generated by the runtime.
  - Normative rule set: See PRD-034 'SSOT: Eligibility vs Registration Execution (Normative)'.

Eligibility Creation Boundary:
- Eligibility MUST be created only when:
  - Decision.scope matches policy.*
  - DecisionVersion is successfully committed
- Eligibility MUST NOT be created by:
  - CompletionPolicy
  - AuditSink
  - UI actions alone

**INT-034-09: Execute Policy Registration (Post-Run Boundary)**
- **Trigger**:
  - post-run lifecycle boundary detects an eligible `PolicyRegistrationRequest`, OR
  - operator submits an eligible `PolicyRegistrationRequest` via CLI
- **Outcome**:
  - Policy Registry is updated by the post-run executor.
  - Normative rule set: See PRD-034 'SSOT: Eligibility vs Registration Execution (Normative)'.
